# 入门课


Friday 是一个脚手架范畴的框架，上一层，我们已经成功新建了一个Friday项目，下面我们来分析如何运行一个Friday项目。



## 实例化Friday

```css dark
import Friday from '@friday/core'
// 实例化Friday
const app = new Friday({
	onInjectConfigBefore() {},
    onInjectConfigAfter() {},
    onInjectRouterAfter(configuration) {},
    onStarted(configuration) {}
})
```
friday提供四个钩子，在不同执行周期执行。

## 注入configuration

```css dark
app.injectConfigurations(configuration[])
```
injectConfigurations接收一个configuration[], 传入多份配置，将根据host自动匹配当前host的配置表。

injectConfigurations必须在app.injectRouters之前注入，否则将会中断启动。因为injectRouters需要使用注入的configuration。

## 中间件（可选）
我们可以选择给friday注册一些中间件，它将作用于之后injectRouters 注入的组件之上。

例如：我们注册一个全局的store，之后所有的component都可以共享这个store 

```css dark

const Context = React.createContext({})

const globalState_middleware: IMiddlewareFC = ({ App, config }) => {
	return {
		App(props) {
			return (
				<Context.Provider>
					<App {...props} />
				</Context.Provider>
			)
		},
		config
	}
}

const middlewares = [
	globalState_middleware
]

app.use(middlewares)

```
通过以上方法，我们可以很方便的扩展。并且将业务进行结偶，可以通过中间件的方式添加国际化等业务。

friday内部的部分逻辑也基于中间件进行扩展，如：@friday/async依赖中间件去注册axios实例。

`@friday/core`提供了`globalState_middleware`的实现, 可以通过`import {GlobalState_middleware} from '@friday/core'`获取

## 注入组件（路由）
```css dark
app.injectRouters(React.FC InjectProps)
```
injectRouters 接收一个React.FC InjectProps 函数组件， InjectProps为

```css dark
InjectProps {
	history: History
	[x: string]: any
}
```


## 启动

```css dark
app.start('#root')
```
一个简单的friday项目就创建完成了。
